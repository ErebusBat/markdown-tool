<objective>
## Goal
Ensure GitHub long-form conversions omit embedded JIRA keys from the output title.

## Purpose
Make GitHub UI-derived markdown more readable by removing unrelated JIRA identifiers while keeping the issue/PR title and link correct.

## Output
Updated GitHub long parser/writer behavior and tests covering JIRA-key exclusion.
</objective>

<acceptance_criteria>

## AC-1: Strip JIRA key from GitHub long titles
```gherkin
Given GitHub long-form UI input that includes a JIRA key like "[HQ-13237] Update rack to 2.2.22 #250"
When the tool converts it to markdown
Then the output title omits the JIRA key while preserving the issue/PR number and remaining title text
```

## AC-2: Preserve non-JIRA titles
```gherkin
Given GitHub long-form input without any JIRA key in the title
When the tool converts it to markdown
Then the title text is unchanged from current behavior
```

## AC-3: Parser does not treat JIRA keys as org/repo or issue number
```gherkin
Given GitHub long-form UI input with a bracketed JIRA key in the title
When parsing org, repo, title, and number
Then the title excludes the JIRA key and the number remains the GitHub issue/PR number
```

</acceptance_criteria>
